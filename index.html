HTML

Hypertext Markup Language

Think about HTML as the basic of the Web. Like the foundation of a webpage.
It describes the content and the structure of a page. It uses XML Markup in order to be built.
It uses tags and elements in order to be built.

So what is an Element?

It's composed by a tag, a value and it's content.


Some elements can even be Self Closing Elements
<img src="images/tag.gif"/>


So, what's the structure of an HTML File?


  • <!DOCTYPE html>
  • <html>
  • <head>
  • <meta>
  • <title>
  • <body>

<!DOCTYPE html>

<!DOCTYPE html> is a browser flag (an instruction) to the web browser that represents the HTML version used.
(The version of HTML the page was written in.)

<html> tag

The <html> tag informs the browser that it's an HTML document.

It also represents the root of an HTML document, and it's also the container for all other HTML elements
(except for the <!DOCTYPE> tag, mentioned above.)

<head> tag

The <html> tag is a container for all the <head> tags, like a title for the document, scripts, styles, etc.

The following tags can be written inside the <head> tag:

  • <title> (this element is required in an HTML document.)
  • <style>
  • <base>
  • <link>
  • <meta>
  • <script>
  • <noscript>

<meta> tag

Metadata is data about data. (DATACEPTION.)
The <meta> tag provides the browser metadata about the HTML document.
It's not displayed, but it's parsable by machines, like a browser, a search engine, other web services...

Meta elements are usually used to specify certain information about the page, such as...

  • Author
  • Page Description
  • Last modification
  • Keywords
  • Character encoding (eg: charset="utf-8")
  • Among others...

<title> tag

The <title> tag is a required HTML element. It represents the title of the HTML Document.
It may be used to...

  • ...define a title in the browsers tab/window
  • ...provide a title when you add the page to the Bookmarks
  • ...provide a title to be used in a Search Engine's results

<body> tag

The <body> tag defines the document's body. It's the root element of all visible (rendered) elements.
It contains all of the elements of the page, such as HyperLinks, Images, Tables, Lists, etc...


Block / Inline Elements

  • Every HTML element in the HTML Document is represented as a rectangular box with some properties.
  • (padding, border, margin...)

  • Block level elements form their own block, and are formatted vertically one after the other.
  • Inline level elements live inside a containing block (they do not have their own block), and are formatted horizontally.

Let's talk about CSS.

What is CSS?

CSS is a style sheet language used for describing the presentation of a document written in a markup language, like HTML.
It allows the structure (HTML) to be isolated from the styling (CSS.) It describes/determines how the elements are styled, laid out and presented.


What is the structure of a CSS rule?

The selector determines which elements will be affected by the rule.
You can also select two or more elements, using a comma (,)
A declaration is a set of property-value.
Updating this values will update the way the HTML elements will be displayed (will appear on screen.)



Can I use CSS on HTML ids and Classes?

On ids On classes Descendant Selector Child Selector

Applying CSS on IDs

In order to apple an CSS Rule to an element represented by an HTML id, you use a hash symbol (#)
Here's an example:

HTML Code: <p id="border"> Lorem Ipsum </p>

CSS Code:
#border {
    color: blue;
    border: 1px solid blue;
}

Applying CSS on Classes

When applying CSS to HTML Classes you use a dot symbol. (.) You can also specify for two exact classes.
The following example will be applied when an element belongs to the exact two classes mentioned on the CSS.

HTML Code:
<h1 class="border header"> HTML Styles </h1>

CSS Code:
.border.header {
    color: blue;
    border: 1px solid blue;
}

Descendant Selector

Imagine you want to apply a CSS Rule to certain elements inside other elements. How do you do that?

<h1 class="title">
    <p>
        <ul>
            <li>Java</li>
            <li>JavaScript</li>
        </ul>
    </p>
</h1>

The following CSS Rule will apply to all <li> elements, inside the title class:

.title li {
    border: 1px solid red;
}

Child Selector

You use a Child Selector to specify an element which is a direct child of another element, like a class.
Here's an example:

<h1 class="title">
    <p>
        <ul class="title">
            <li>Java<li>
        </ul>
    </p>
</h1>

The following CSS Rule will be applied to the <li> element ("Java"), as it's a direct child of the class title.

.title > li {
    border: 1px solid red;
}

Static Positioning

Static Positioning is the default positioning accordingly to the browser.

<style>
    .box {
        display: inline-block; /* layout boxes horizontally */
        background: red;
    }
    #two {
        background: green;
    }
</style>

<div class="parent">
    <div id="one" class="box">1</div>
    <div id="two" class="box">2</div>
    <div id="three" class="box">3</div>
</div>

Relative Positioning

Relative Positioning means the new position is relative to the default one. Think of it as a shifted default position.

Other elements won't change, as the "default" position to the changed element remains the same.

<style>
.box {
    display: inline-block; /* layout boxes horizontally */
    background: red;
}
#two {
    position: relative;
    background: green;
    left: 20px;
    top: 10px;
}
</style>

Absolute Positioning

Absolute Positioning means the position of the element is coded. It means there's no default position (or absolute position acts as a default),
and other elements may change.

<style>
    .box {
        display: inline-block; /* layout boxes horizontally */
        position: relative; /* use as a container for positioning child elements */
        background: red;
    }
    #two {
        position: absolute;
        background: green;
        left: 20px;
        top: 10px;
    }
</style>

z-index

Positioned elements can be moved to the front and the back using the z-index property.

A positive index will move the element to the front, a negative will move the element to the back.

<style>
    .box {
        display: inline-block; /* layout boxes horizontally */
        position: relative; /* use as a container for positioning child elements */
        background: red;
    }
    #two {
        position: absolute;
        z-index: -100;
        background: green;
        left: 20px;
        top: 10px;
    }
</style>
